# FocusTimer Agent Notes

## Project Shape

FocusTimer is a macOS 14+ SwiftUI menu-bar app generated by XcodeGen. `project.yml` is the source of truth for project structure; regenerate `FocusTimer.xcodeproj` with `xcodegen generate` after changing sources/resources/plist/entitlements layout.

## Commands

- Generate project: `xcodegen generate`
- Debug build: `xcodebuild -project FocusTimer.xcodeproj -scheme FocusTimer -configuration Debug build`
- Test: `xcodebuild test -project FocusTimer.xcodeproj -scheme FocusTimer -configuration Debug -destination 'platform=macOS'`
- Release/install details live in `PUBLISH.md`.

## Sharp Edges

- macOS has no public setter API for Focus mode. `INFocusStatusCenter` is read-only; Focus writes must stay behind `/usr/bin/shortcuts run` through `ProcessRunner`.
- Shortcut names are fixed and UI-read-only: `开始专注` and `关闭专注`. Keep `FocusTimerModel.defaultEnableShortcut`, `FocusTimerModel.defaultDisableShortcut`, `LiveShortcutInstaller.defaultEnableName`, `LiveShortcutInstaller.defaultDisableName`, and `ShortcutRole.bundledResourceName` in sync.
- Bundled Shortcut resource filenames are `FocusTimer/Resources/Shortcuts/开始专注.shortcut` and `FocusTimer/Resources/Shortcuts/关闭专注.shortcut`. macOS imports signed `.shortcut` files using the filename as the visible Shortcut name.
- `FocusTimerModel` removes old editable Shortcut UserDefaults keys on init. Do not reintroduce editable names unless the bundled import flow is redesigned too.
- Full-screen rest reminders default on via `FocusTimer.showRestReminder`; their 5/15/30 minute countdown is local to `RestBreakTimerModel` and must not mutate the main focus timer state.
- App Sandbox is disabled in `FocusTimer/Resources/FocusTimer.entitlements`. Re-evaluate `/usr/bin/shortcuts`, notifications, and Focus status behavior before enabling sandbox or preparing App Store distribution.
